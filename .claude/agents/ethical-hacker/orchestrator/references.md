## Primary Sources

### Penetration Testing Methodologies

#### PTES (Penetration Testing Execution Standard)
- **Source**: http://www.pentest-standard.org/
- **Description**: Seven-phase methodology covering pre-engagement through reporting
- **Phases**:
  1. Pre-engagement Interactions
  2. Intelligence Gathering
  3. Threat Modeling
  4. Vulnerability Analysis
  5. Exploitation
  6. Post-Exploitation
  7. Reporting

#### OWASP Testing Guide
- **Source**: https://owasp.org/www-project-web-security-testing-guide/
- **Version**: v4.2 (latest)
- **Description**: Comprehensive web application security testing framework
- **Key Resource**: https://owasp.org/www-project-web-security-testing-guide/v41/3-The_OWASP_Testing_Framework/1-Penetration_Testing_Methodologies

#### NIST SP 800-115
- **Source**: https://csrc.nist.gov/publications/detail/sp/800-115/final
- **Description**: Technical Guide to Information Security Testing and Assessment
- **Application**: Federal and enterprise security assessments

### Framework Comparisons

#### PTES vs OWASP Analysis
- **Source**: https://www.pixelqa.com/blog/post/penetration-testing-frameworks-ptes-vs-owasp-analysis
- **Key Insight**: PTES provides broader operational scope; OWASP offers deeper web-specific guidance
- **Recommendation**: Blend methodologies for comprehensive coverage

### Industry Best Practices 2025

#### DeepStrike Methodology Guide
- **Source**: https://deepstrike.io/blog/penetration-testing-methodology
- **Key Points**:
  - Modern pentesting must address cloud-native environments
  - API-heavy architectures require specialized testing
  - AI-powered threats demand new defensive strategies

#### IBM Penetration Testing Insights
- **Source**: https://www.ibm.com/think/insights/pen-testing-methodology
- **Key Points**:
  - Mature testing requires "polymethodologist" approach
  - Blend PTES lifecycle with OWASP technical depth
  - Integrate MITRE ATT&CK threat intelligence
